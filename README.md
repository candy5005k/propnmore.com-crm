## 2. Database Setup

1. Open **phpMyAdmin**
2. Create database: `crm_lsr`
3. Import `database.sql`
4. That creates all tables + two admin accounts

---

## 3. Edit config.php

Open `config.php` and fill:

```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'crm_lsr');
define('DB_USER', 'your_db_user');
define('DB_PASS', 'your_db_password');
define('BASE_URL', 'https://yourdomain.com/crm');   // No trailing slash

define('OTP_EMAIL', 'lakshsiddhrealty@gmail.com');  // OTPs go here
```

Update Sheet IDs if Meta/Google leads are in a different Google Sheet:
```php
define('SHEET_META_ID',  'your_meta_sheet_id');
define('SHEET_META_TAB', 'LSR Leads 2026');
define('SHEET_GOOGLE_TAB', 'Google Leads');
define('SHEET_SALES_TAB',  'Sales Team');
```

## 4. Google Sheets Auth (token.json)

The `token.json` file was already generated by your Python script (`cnp_leads.py`).  

**Just copy `token.json` to the CRM root folder** (same folder as `config.php`).  
Also copy `credentials.json` to the same folder.

The CRM will auto-refresh the token when it expires.

## 7. Sales Manager Flow

1. Sales manager visits `/register.php` → fills name, email, password
2. Admin receives notification email
3. Admin goes to **Users** → clicks **Activate**
4. Sales manager logs in with email + password
5. Sales manager can only see leads **assigned to them**

## 8. Syncing Google Sheets

1. Login as admin → **Sync Sheets** in sidebar
2. Select which tabs to sync → click **Run Sync Now**
3. New leads are imported; existing ones (matched by mobile) are skipped
4. Check sync log for confirmation

**Column mapping (default):**
```
Column A: Sr/Row ID
Column B: Project Name
Column C: First Name
Column D: Last Name
Column E: Mobile
Column F: Email
Column G: Preference
<!-- Column H: Date // 2nd column B -->
Column I: Time
Column J: Page URL
```
If your sheet has different columns, edit the `syncMeta()` / `syncGoogle()` methods in `includes/sheets.php`.

## 11. ✅ Notification System — Sales Manager Lead Assignment Alerts

When an admin assigns a lead to a sales manager (via lead detail quick-update, lead edit page, or new lead creation), an **in-app notification** is instantly created for that sales manager.

**Features:**
- 🔔 Notification bell in the topbar with **unread count badge** (red pulse animation)
- Click bell to see notification dropdown panel
- Each notification shows: title, message, timestamp (human-readable "2h ago")
- Click a notification → goes to lead detail page
- "Mark all read" button
- Notifications auto-refresh every 30 seconds (AJAX polling)
- Database table: `notifications` (user_id, type, title, message, lead_id, is_read)

**Files changed:** `config.php`, `header.php`, `footer.php`, `lead_detail.php`, `lead_edit.php`, `leads_new.php`, `database.sql`

## PHPMailer (Recommended for OTP)

Install PHPMailer via Composer or download manually.  
Then replace the `sendMail()` function in `config.php`:

```php
function sendMail(string $to, string $subject, string $body): bool {
    $mail = new PHPMailer\PHPMailer\PHPMailer(true);
    $mail->isSMTP();
    $mail->Host       = 'smtp.gmail.com';
    $mail->SMTPAuth   = true;
    $mail->Username   = 'lakshsiddhrealty@gmail.com';
    $mail->Password   = 'your_app_password';  // Gmail App Password
    $mail->SMTPSecure = 'tls';
    $mail->Port       = 587;
    $mail->setFrom('noreply@propnmore.com', 'CRM LSR LEADS 2026');
    $mail->addAddress($to);
    $mail->isHTML(true);
    $mail->Subject = $subject;
    $mail->Body    = $body;
    $mail->send();
    return true;
}
```
